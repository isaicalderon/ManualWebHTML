    1. Entidades

      Una entidad es la representación de una tabla en la base de datos. Cada instancia de esta, es una fila que se puede consultar, eliminar, modificar y agregar. Debido a que las entidades son objetos, estos pueden tener otros objetos dentro de ellas. Ejemplo:
      public class Auto{
      private Integer identificador;
      private Marca idMarca;
      private Modelo idModelo;
      }
      Se pueden ver los objetos, estos objetos representan toda la información de una entidad creada anteriormente, esta entidad contiene datos correspondientes a su tabla. Para simplificar, una entidad contiene los datos de su tabla, pero si un campo hace referencia a otra tabla que ya tiene una entidad, y en un futuro se puede ocupar un campo aparte de su identificador, se utiliza la entidad de esta misma, ya que aquí podremos guardar datos de la tabla y datos de su tabla relacionada.

      Ejemplo de consulta: SELECT AU.*, MA.DESCRIPCIONMARCA, MO.DESCRIPCIONMODELO FROM AUTO AU LEFT JOIN MARCA MA ON AU.IDMARCA = MA.IDMARCA LEFT JOIN MODELO MO ON AU.IDMODELO = MO.MODELO AND AU.MARCA = MO.MARCA;

      En esta consulta, se manda a pedir la descripción de la marca y el modelo en donde el id de cada uno en el automóvil sea igual en cada una de sus tablas, en el caso de modelo se tiene que tomar en cuenta la marca también porque los modelos vienen de una marca de automóviles.

    2. Persistencia

      La persistencia en acceso a datos se utiliza para poder mandar consultas a la base de datos creando capas dentro de la misma, aquí se crean interfaces DAO, clases DAO jdbc y la fachada.

      ¿Qué es un DAO?
      Un Data Access Object (DAO) es un objeto que provee una interfaz abstracta a algunos tipos de base de datos u otros mecanismos de persistencia. Esta guía las llamada de la aplicación a la capa de persistencia, el DAO provee algunas operaciones de datos específicas sin exponer las llamadas de la base de datos. Esto está considerado como buena práctica por Sun Microsystems.

      ¿Qué es un DAO jdbc?
      Un jdbc utiliza los métodos de la interfaz de DAO de su tabla y un DAO genérico, el genérico puede contener las operaciones normales (guardar, modificar, eliminar, obtenerTodos y obtenerPorID), aquí va la lógica de la llamada con la conexión a la base de datos, aquí se generan los query para llamar a la base de datos, estos son representados por String, en cada método se maneja la lógica correspondiente a cada tipo de consulta. También se ven los métodos creados en cada DAO que hayamos creado aparte del genérico, las operaciones especiales para las tablas.

      ¿Qué es una fachada?
      El patrón de fachada es un patrón de diseño de software, se usa comúnmente en la programación orientada a objetos. La fachada, como su nombre lo dice, es un objeto que sirve como cara a mostrar al cliente, esta enmascara el código más complejo que está debajo de ella. Los desarrolladores utilizan este patrón cuando el diseño del código de un proyecto es complejo o difícil de entender.

      ¿Cómo sé cuáles métodos genéricos debo de utilizar? ¿Y si ocupo un procedimiento o función que no es genérico?
      Es fácil, para encontrar los procedimientos, primero debemos ir a la base de datos, buscar en PACKAGES, PROCEDURES y FUNCTIONS cuáles son los que necesitamos dependiendo de nuestra tabla, por lo general los nombres llevan esta nomenclatura: “TIPO”_”ACCION”_”TABLA” Los que se necesitan en este manual son los siguientes:

      “PRO_GET_ALL_ALUMNOS” – Obtiene todos los alumnos de la tabla.
      “PRO_GET_ALUMNO” – Obtiene un alumno por su ID.
      “PRO_EDIT_ALUMNO” – Edita la información de un alumno.
      “PRO_DEL_ALUMNO” – Elimina un alumno por su ID.
      “FUN_ADD_ALUMNO” – Agrega a un alumno y le asigna un ID.
